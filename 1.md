  RTASR 实时语音转写 Bug 记录（2026-04-22）

  Bug 1：大模型版降级到标准版

  文件：backend/src-tauri/src/speech_eval/rtasr.rs

  现象：用户设置选择大模型版（use_llm=true），连接失败后自动 fallback 到标准版执行。

  原因：代码设计了"失败自动降级"逻辑，两个版本都放进数组轮流尝试。

  修复：移除 fallback，按 use_llm 直接选择一个版本，失败直接报错。

  ---
  Bug 2：大模型版鉴权 baseString 编码方式错误

  文件：backend/src-tauri/src/speech_eval/auth.rs → build_rtasr_llm_url

  现象：大模型版 WebSocket 鉴权失败。

  原因：baseString 拼接时使用了原始值（未编码），但官方文档和 Python demo 要求对 key 
  和 value 都做 percent-encode（urllib.parse.quote(x, safe='')）再拼接。
                                                                                     
  修复：baseString 改为对 key/value 都做 RFC 3986 percent-encode 后再拼接。          
  
  ---                                                                                
  Bug 3：大模型版 utc 时间格式错误        
                                                                                     
  文件：backend/src-tauri/src/speech_eval/auth.rs → build_rtasr_llm_url
                                                                                     
  现象：大模型版鉴权签名不正确。                                                     
                                                                                     
  原因：使用 %:z 格式输出 +08:00（含冒号），再手动 replace 处理，经过 URL 编码后 +   
  变成 %2B，导致签名计算与 URL 参数不一致。
                                                                                     
  修复：改用 %z 直接输出 +0800（无冒号），无需二次处理。                             
  
  ---                                                                                
  Bug 4：大模型版 lang 参数传值错误       
                                                                                     
  文件：backend/src-tauri/src/speech_eval/rtasr.rs
                                                                                     
  现象：传入 lang=es，大模型版不支持此值。                                           
                                                                                     
  原因：大模型版只支持 autodialect（中英+方言）和 autominor（37语种），不接受 ISO    
  语言代码。                              
                                                                                     
  修复：大模型版固定使用 autominor（支持西班牙语等37语种），标准版继续用 es。        
  
  ---                                                                                
  Bug 5：标准版 data 字段 base64 解码错误（核心 Bug）
                                                                                     
  文件：backend/src-tauri/src/speech_eval/rtasr.rs → extract_text_from_std_data
                                                                                     
  现象：标准版连接成功、session started、音频发送完毕，但识别结果始终为空。          
                                                                                     
  原因：代码注释和实现都将 data 字段当作 base64 编码的 JSON 来解码，但标准版 API     
  返回的 data 字段是普通 JSON 字符串（文档示例中可以明确看到）。base64
  解码失败导致所有识别结果被静默丢弃。                                               
                                          
  修复：extract_text_from_std_data 改为直接 serde_json::from_str(data) 解析，同时将  
  is_final 判断（type=="0" 或
  ls==true）合并到函数返回值中，移除调用处重复的二次解码逻辑。

  Bug 6：TLS 库导致大模型版 WebSocket 握手失败（待验证）
                                                                                     
  文件：backend/src-tauri/Cargo.toml
                                                                                     
  现象：大模型版始终报 httparse error: invalid response status，curl                 
  测试服务器返回非标准 HTTP 响应。                                                   
                                                                                     
  原因（推测）：rustls-tls-webpki-roots 在 TLS 握手时的 ALPN 行为与 Python           
  websocket-client 不同，导致服务器响应异常。
                                                                                     
  修复：改用 native-tls（macOS 使用系统 SecureTransport，Android 使用系统            
  OpenSSL），行为与 Python 一致。
                                                                                     
  ▎ 注：大模型版连接问题根本原因尚未完全确认，native-tls                             
  ▎ 切换是当前推测方案，待测试验证。
